## Session 13 — Phase D: /reconcile skill + drift audit (2026-04-24, evening)

Built a custom `/reconcile` skill at `.claude/skills/reconcile/` — complements `/lint` (shape checks) with deterministic semantic-drift checks.

**What it checks:**

1. Date integrity — `created` / `updated` / `valid_from` / `learned_at` must be `YYYY-MM-DD` and ordered (`created ≤ updated`; `valid_from ≤ updated`).
2. ADR supersede-chain reciprocity — if X has `supersedes: [[Y]]`, Y must have `superseded-by: [[X]]`.
3. ID / alias collisions — no alias on one note may equal an id or alias of a different note.
4. Stale drafts — `status: draft` older than 14 days (tunable).
5. Duplicate tags within a single note.

Deterministic only — semantic contradiction detection would need an LLM pass and is out of scope. Read-only; prints a categorised report; exit 1 on findings.

**First-run findings (3 real issues):**

- `wiki/entities/albaha-center-org.md`, `albaha-regional-admin.md`, and `mhrsd-dev-sector-esystems.md` all had `updated: 2026-04-23` but `created: 2026-04-24`. Wrong convention — `updated` should be when the *vault note* was last written (today), not when the source content was captured. Source-capture date belongs in `valid_from` / `learned_at`, which were already correct.

**Fixed:** set `updated: 2026-04-24` on the three entity notes. Re-ran `/reconcile` + `/lint` — both clean.

**Also updated `_CLAUDE.md`:** the Skills section now lists the three built skills (`regenerate-indexes`, `lint`, `reconcile`) with run commands, and explicitly marks `log-session` as deferred because a deterministic scaffold has low ROI compared to the other checks.
