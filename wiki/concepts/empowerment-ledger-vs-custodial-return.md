# Empowerment Ledger vs Custodial Return

## The pair

Two mirrors the same database can emit. They measure opposite things.

**The custodial monthly return** — the mandatory ministry statistical aggregate (census, طي القيد, clinical events, staff census), reported per gender. Its clinical spine is the **motor-ability 3-tier classification** — طريح فراش (bedridden) / مقعد (chair-bound) / متحرك (mobile) — and that tier is **the financial mechanism**: care-staffing ratios and funding tiers scale by dependency (bedridden = top-funded). This is *function, not diagnosis*, but it rewards keeping people maximally dependent and resident.

**The empowerment ledger** — the counterweight the same DB emits: functional-improvement deltas (طريح فراش → متحرك → discharged-home), skills acquired, **family-reintegration exits WITH destination**, and aftercare follow-ups.

## Why the pair is load-bearing

The custodial return literally **cannot see empowerment.** An improving resident appears only as a *lower count in a lower-funded tier* plus a **reasonless طي القيد line** — empowerment reads as institutional shrinkage and revenue loss. An empowering exit (returned to family) is indistinguishable from a death or transfer because the exit carries no destination.

The **motor-tier funding trap** is the single most-corrosive anti-pattern: a direct financial incentive against the person improving — the same pathology class as the مادة الصرف severity tiers ([[basira-v4-disability-empowerment-model]]), here generalized into a *funding-mechanism* instance. The «فخ الإعانة» benefit-trap is its demand-side twin ([[family-fear-barrier]]).

The ledger reframes success: a center becomes measurable on the people it **empowered out**, not only the people it **holds**. This operationalizes Ahmad's frame that the win is the file closing for the right reason. The compliance return earns its keep only when shipped *with* the ledger — never STAT alone.

## North-star anchor

The ledger's headline KPI is the **residential→non-residential transition rate (ت-1-1), 2019 baseline = 2%** — the single most concrete "before" datapoint in the vault ([[community-integration-project]], [[mhrsd-kpi-landscape-2024-2026]]). The ledger is what moves that number from a half-story baseline into a tracked, destination-verified climb. Pair any discharge metric with a post-discharge integration-quality check (readiness in [[integration-readiness-classification]]).
